## ЧАСТЬ 1. Проверить (просто потыкать)

По каждому пункту в конце поставь **✅ работает** или **❌ сломано + скрин**.

### ① Pipeline — «задача с доказательством»

Это новая кнопка-сценарий: ведёт от задачи до готового отчёта по шагам.

1. Убедись, что выбран провайдер **Claude (API)** или **Gemini API** (вверху справа в композере). Если стоит CLI — переключи, иначе кнопки не будет.
2. В поле ввода чата слева найди кнопку **▶ Pipeline**. Нажми.
3. Откроется окошко с тремя полями. Заполни:
   - **Цель:** `исправить ошибки типов в проекте`
   - **Границы:** можно пусто
   - **Как проверим:** `npm run type`
4. Кнопка **«Сформировать план»** должна загореться только когда заполнены «Цель» и «Как проверим». Нажми её.

   👉 Должна появиться **полоска сверху над вводом**: «Pipeline · 2/5 · План». Агент пишет план, файлы пока не трогает.
5. На полоске нажми **«План OK → Выполнить»** → полоска станет «3/5 · Выполнение», агент начнёт править.
6. Нажми **«→ Проверка»** → полоска «4/5 · Проверка» + цветной значок: 🟢 / 🟡 / 🟢.
7. Когда 🟢 — нажми **«→ Proof Pack»** → справа откроется готовый HTML-отчёт, полоска пропадёт.

**Что проверяем:** полоска всегда показывает «N/5», переходы работают, в конце открывается отчёт.

➡️ Результат: **✅ / ❌ + скрин**

---

### ② «В скилл» — сохранить прогон

1. Слева в меню открой вкладку **«Прогоны»**.
2. Кликни любой завершённый прогон в списке.
3. Сверху рядом с кнопкой «🔏 Proof Pack» появилась новая — **«⭐ В скилл»**. Нажми.

   👉 Должна выскочить надпись «✓ Скилл сохранён».
4. Открой пикер скиллов (значок **🎭** в поле ввода) — там должен появиться новый скилл (если нет — нажми обновить).

➡️ Результат: **✅ / ❌ + скрин**

---

### ③ Авто-проверка после правок

Проверяем, что агент сам ловит свои ошибки.

1. Провайдер — снова **Claude (API)** или **Gemini API**.
2. Напиши агенту: `добавь в файл src/lib/pricing.ts строку: const x: number = "это строка"`
3. Прими правку (агент изменит файл).

   👉 На следующем ответе агент должен **сам написать**, что нашёл ошибку типов, и **починить её** — без твоих напоминаний.

➡️ Результат: **✅ / ❌ + скрин**

## ЧАСТЬ 2. Дизайн (твоё)

Экран **Pipeline** (тот, что в пункте ① — окошко с тремя полями + полоска сверху) собран на функцию, но выглядит черновым. Причеши под наш стиль (Atelier), как остальные окна и панели.

**Что красивим:**
- Окошко с тремя полями (бриф) — отступы, шрифты, кнопка.
- Полоску «Pipeline · N/5 · шаг» — компактная, читаемая; цвет значка проверки (зелёный/жёлтый/красный).
- Проверь в **тёмной и светлой** теме.

**Файлы:** стили — `src/styles/*.css` (ищи `gg-pipeline`). Вёрстка — `src/components/PipelineWizard.tsx` и `PipelineBanner.tsx` (меняй только классы/разметку, логику не трогай). `electron/` — не лезь.

**Перед PR прогони:**
```
npm run type        (должно быть 0 ошибок)
npm run test:fast   (всё зелёное)
npm run build       (собирается)
```
Дальше как обычно — свой форк, коммиты `(rayner)`.
