## Часть 1 — Проверка (QA, прокликать в запущенном приложении)

Запусти `npm run dev`, открой проект **verstak**. Прогони 3 сценария, по каждому отметь ✅/❌ + скрин если сломано.

### 1.1 Pipeline Brief→Proof (флагман)
1. В композере чата нажми **▶ Pipeline** (кнопка слева от Send; видна только на API-провайдере — если её нет, переключись на Claude API / Gemini API).
2. В модалке заполни 3 поля: **Цель** (напр. «исправить ошибки типов в src»), **Границы** (опц.), **Как проверим / DoD** (напр. «npm run type»). Кнопка «Сформировать план» должна стать активной только когда заполнены Цель и DoD.
3. Нажми «Сформировать план» → должен появиться **sticky-баннер** «Pipeline · 2/5 · План», агент идёт в read-only режиме (файлы НЕ трогает) и создаёт план.
4. Нажми «**План OK → Выполнить**» → баннер «3/5 · Выполнение», агент применяет правки.
5. Нажми «**→ Проверка**» → баннер «4/5 · Проверка» со **светофором**: 🟢 прошло / 🟡 не подтверждено / 🔴 красные. При 🟡/🔴 — кнопка «Дожать проверку».
6. На 🟢 нажми «**→ Proof Pack**» → соберётся HTML-Proof, откроется в preview-панели, баннер исчезнет (pipeline completed).
- **Ожидание:** на каждом шаге баннер показывает N/5; план не пишет файлы; Proof открывается. Кнопка ▶ Pipeline дизейблится на CLI-провайдере с тултипом.

### 1.2 Skill Capture
1. Открой любой завершённый прогон (вкладка **Прогоны** → выбери прогон).
2. Рядом с кнопкой «🔏 Proof Pack» нажми «**⭐ В скилл**».
3. **Ожидание:** сообщение «✓ Скилл «…» сохранён», и в `~/.verstak/skills/` появился `<id>.md` (frontmatter + тело). Открой пикер скиллов 🎭 — новый скилл там есть (после refresh).

### 1.3 Diagnostic Loop v2 (авто-tsc)
1. Попроси агента (API-провайдер) внести в любой `.ts` файл правку с ОШИБКОЙ типа (напр. «добавь в src/lib/pricing.ts строку `const x: number = "строка"`»).
2. Прими правку.
3. **Ожидание:** на следующем ходу агент САМ получает ошибку tsc в контекст («авто-проверка типов нашла ошибки…») и чинит её, не дожидаясь напоминания. (Отключается настройкой `diagnostic_loop=false`.)

**Что прислать:** по каждому 1.1–1.3 — ✅/❌ + где сломалось (скрин/текст).

## Часть 2 — Дизайн-полиш Pipeline-UI

`PipelineWizard.tsx`, `PipelineBanner.tsx` + секции `.gg-pipeline-*` в `src/styles/layout.css` собраны функционально, **дизайн черновой**. Приведи к стилю приложения (Atelier), как остальные модалки/баннеры:
- Модалка брифа (`gg-pipeline-wizard`) — отступы, типографика 3 полей, активная/неактивная кнопка.
- Sticky-баннер (`gg-pipeline-banner`) — компактный, читаемый «N/5 · шаг», цвет светофора Verify (`.gg-pipeline-verify.is-pass/.is-warn/.is-fail`).
- Тёмная И светлая темы.

**Можно:** `src/styles/*.css`, `className`/разметку в `PipelineWizard.tsx`/`PipelineBanner.tsx`. **Нельзя:** логику (хэндлеры/стейт/`window.api`), `electron/`.

**Перед PR:** `npm run type` (0 ошибок) · `npm run test:fast` (зелёные) · `npm run build`. Работай в своём форке как обычно, помечай `(rayner)`.
