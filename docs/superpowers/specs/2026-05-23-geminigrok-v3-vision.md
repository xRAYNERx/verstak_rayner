# GeminiGrok V3 — Vision Spec
**Дата:** 2026-05-23  
**Контекст:** Brainstorm сессия ai-lab → передача в bos-hq

---

## Одна фраза

**GeminiGrok V3 = Manus для маркетингового агентства.**

Пишешь задачу на обычном языке — система делает под ключ.

---

## Что это значит

Не чат-ассистент. Не инструмент помощи.  
**Агент-исполнитель** — получает задачу, планирует шаги, выполняет, возвращает результат.

Примеры реальных задач которые должна закрывать система:
- «Собери отчёт по клиенту, оформи в презентацию»
- «Настрой рекламу в Яндекс Директ по чеклисту»
- «Вот конкуренты — сделай анализ по офферам, дай отчёт»
- «Собери аналитику по клиенту за месяц»

---

## Ключевой принцип: контекст по требованию

Система **не хранит базу клиентов**.  
Маркетолог говорит «Стимбаланс» — агент сам находит нужный контекст (файл агента, кабинет, историю) и использует его.  
Клиенты приходят и уходят. Система адаптируется без перенастройки.

## 12 блоков возможностей

| # | Блок | Что делает в агентстве |
|---|---|---|
| 1 | Управление файлами | Отчёты, КП, брифы, аудиты — по стандартам агентства |
| 2 | Управление браузером | Конкурентный анализ, парсинг офферов, Авито, hh.ru |
| 3 | Кодинг | Скрипты для Метрики, обработка данных, автовыгрузки |
| 4 | Работа с API | Яндекс Директ, Метрика, Битрикс24, VK Ads |
| 5 | Автоматизация | Настройка РК по чеклисту, минусация, чистка площадок |
| 6 | Мониторинг | Упал CTR, кончается бюджет, отклонили объявление |
| 7 | Мультимедиа | Презентации, PDF, графики, генерация креативов |
| 8 | Планирование | Еженедельный отчёт, утренний обход, повторяющиеся задачи |
| 9 | Голос | Говоришь задачу — получаешь результат |
| 10 | Мультиагентность | Несколько задач параллельно разными агентами |
| 11 | Память | Система умнеет от каждой задачи, накапливает ДНК агентства |
| 12 | Telegram | Задача пришла в чат → агент подхватил и сделал |

---

## Отличие от Manus

| Manus | GeminiGrok V3 |
|---|---|
| Общий агент, начинает с нуля | Знает инструменты агентства |
| Браузер как единственный путь | Прямые API (быстрее, надёжнее) |
| Нет памяти между задачами | Накапливает что работает |
| Не знает что такое хорошая реклама | Чеклисты и стандарты агентства внутри |

## Архитектурные решения (из предыдущей сессии)

- **Скиллы** — с сервера через API (`/api/skills/{id}/context`), не локально
- **Google Sheets** — тот же service account из `/opt/los/creds.json`
- **Telegram** — новый бот через @BotFather
- **SSH** — отдельный пользователь `gemini-agent` с whitelist команд
- **14 сотрудников** — каждый со своим API key
- **Артефакты** — HTML + DOCX
- **Bitrix пилот** — БАУМЕХ первый
