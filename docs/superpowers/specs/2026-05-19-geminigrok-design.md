# Verstak — Design Spec

**Дата:** 2026-05-19
**Автор:** Павел Фролов
**Статус:** Draft v1

---

## 1. Что это

Десктоп-приложение для AI-кодинга. Чистый аналог Cursor / ClawCode / Claude Code Desktop, но:

- Работает на пользовательских API-ключах (Gemini в первую очередь, затем Claude и GPT)
- Без подписок и серверной части — пользователь платит провайдеру модели напрямую
- Chat-first интерфейс (как Claude Code), без встроенного редактора кода
- Хобби-проект с потенциалом монетизации

## 2. Что мотивирует

- У Павла есть Gemini Ultra подписка → бесплатный API-ключ через AI Studio
- Antigravity (Google) и аналоги блокируются по гео в РФ
- Agent Studio покрывает работу со скиллами Павла, но не подходит для чистой работы с проектами
- Хобби — заявленная цель Павла найти "что-то делать руками"

## 3. Ключевое отличие от Agent Studio

| | Agent Studio | Verstak |
|---|---|---|
| Зачем | Личная операционная система | Кодинг по проектам |
| Главная сущность | Скилл / чат | Проект |
| Память | LOS / BOS / Sheets | История чатов в проекте |
| Аудитория | Только Павел | Любой разработчик |
| Сложность | Высокая (флоу, задачи, скиллы) | Минимальная |

## 4. Сценарий использования

1. Пользователь открывает Verstak
2. Выбирает или открывает папку проекта
3. В сайдбаре видит дерево файлов
4. Пишет задачу в чат: "добавь авторизацию через JWT"
5. AI читает нужные файлы, предлагает diff, ждёт подтверждения
6. Пользователь жмёт "Принять" → файлы изменяются
7. AI запускает команды в встроенном терминале при необходимости

## 5. Архитектура

### 5.1 Стек

- **Electron + Vite** — десктоп оболочка
- **React + TypeScript** — UI
- **better-sqlite3** — хранение истории чатов и настроек
- **@google/genai** — Gemini SDK
- **@anthropic-ai/sdk** — Claude (добавим во второй итерации)
- **openai** — GPT (третья итерация)
- **node-pty** — встроенный терминал
- **simple-git** — git операции

### 5.2 Модули

```
verstak/
├── electron/
│   ├── main.ts              # Electron entry, окно
│   ├── preload.ts           # Безопасный мост renderer↔main
│   ├── ipc/
│   │   ├── projects.ts      # Открыть/переключить проект
│   │   ├── files.ts         # Чтение/запись файлов
│   │   ├── terminal.ts      # node-pty wrapper
│   │   └── ai.ts            # Маршрутизация в провайдер
│   ├── ai/
│   │   ├── types.ts         # Единый интерфейс ChatProvider
│   │   ├── gemini.ts        # Реализация для Gemini
│   │   └── tools.ts         # File/terminal tools для AI
│   └── storage/
│       ├── db.ts            # SQLite инициализация
│       └── settings.ts      # API ключи, выбранная модель
├── src/                     # React renderer
│   ├── App.tsx
│   ├── components/
│   │   ├── Sidebar.tsx      # Дерево файлов + переключатель моделей
│   │   ├── Chat.tsx         # Чат с AI
│   │   ├── DiffView.tsx     # Просмотр предложенных изменений
│   │   ├── Terminal.tsx     # xterm.js терминал
│   │   └── Settings.tsx     # Окно настроек (API ключи)

│       └── projectStore.ts  # Zustand state (открытый проект, чат)
└── docs/
```

### 5.3 Главная абстракция — ChatProvider

```typescript
interface ChatProvider {
  id: 'gemini' | 'claude' | 'gpt'
  name: string
  models: string[]
  send(messages: Message[], tools: Tool[]): AsyncIterable<ChatEvent>
}
```

Все три провайдера реализуют этот интерфейс. UI вообще не знает с какой моделью говорит.

### 5.4 Поток данных

```
Renderer (Chat.tsx)
    ↓ IPC: send-message
Main (ipc/ai.ts)
    ↓ выбор провайдера по settings.activeModel
ai/gemini.ts → @google/genai
    ↓ stream chunks
IPC events → Renderer
    → Чат обновляется в реальном времени
    → AI вызывает tools (read_file, write_file, run_command)
    → Tool calls идут обратно в Main, выполняются, результат летит обратно
```

## 6. Что в MVP (первые 2-3 недели)

**Включено:**
- Открыть папку проекта
- Сайдбар с деревом файлов (без редактирования внутри)
- Чат с Gemini 2.5 Pro
- Tools: read_file, write_file, list_directory, run_command
- Diff-просмотр перед применением изменений
- Сохранение API-ключа в SQLite
- История чатов по проектам

**НЕ включено в MVP (отложено):**
- Claude и GPT провайдеры
- Multi-window / split view
- Git UI (только команды через терминал)
- Подсветка синтаксиса в diff-просмотре
- Распространение как .exe (запуск через `npm run dev`)
- Telemetry / analytics
- Auto-update

## 7. Риски и ограничения

- **Размер контекста.** Gemini 2.5 Pro даёт 1M токенов — этого достаточно для большинства проектов. Но в больших репозиториях нужна стратегия "что слать в контекст". В MVP — посылаем только файлы которые AI явно запросил через tool.
- **Безопасность tools.** AI может удалить файлы или выполнить опасную команду. В MVP — always-confirm на write_file и run_command.
- **Streaming.** Gemini SDK поддерживает streaming, но обработка tool calls в стриме сложная. В MVP — sync режим, потом стрим.
- **Кросс-платформенность.** Стартуем с Windows (где работает Павел). macOS/Linux — потом.

## 8. Что после MVP

- Claude и GPT провайдеры (через тот же `ChatProvider`)
- Git UI (визуализация изменений, коммиты из приложения)
- Inline-редактирование файлов через Monaco
- Скиллы / пресеты (системные промпты для разных задач: рефакторинг, тесты, документация)
- Распространение — `.exe` через electron-builder
- Возможность платной версии (свой API-прокси, синхронизация настроек)

## 9. Критерии готовности MVP

- [ ] Открыть папку C:\Users\Pavel\some-project и увидеть её дерево
- [ ] Написать "опиши что в этом проекте" → Gemini читает файлы и отвечает
- [ ] Написать "добавь в README раздел Setup" → AI предлагает diff → жму принять → README меняется
- [ ] Написать "запусти npm test" → команда выполняется в встроенном терминале → AI видит результат
- [ ] Закрыть и открыть приложение — история чатов сохранилась

## 10. Открытые вопросы

- Хранить ли API-ключи в plaintext SQLite или через Electron safeStorage? **Решение:** safeStorage (нативное шифрование OS).
- Где хранить историю чатов — глобально или в папке проекта (как `.verstak/`)? **Решение:** в `.verstak/` каждого проекта — пользователь может удалить вместе с проектом, легко шарить настройки.
- Название финальное? `verstak` — рабочее, для домена/публикации может понадобиться что-то получше.
